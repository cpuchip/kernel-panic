# Kernel Panic — Claude Code project context

A popcorn **tower defense** parody, single- and multiplayer. Its own git repo
(`github.com/cpuchip/kernel-panic`), nested in the `scripture-study` workspace at
`projects/kernel-panic/` and gitignored from the parent. Sibling of first-orbit,
deadweight, and the chips suite; same house harness and discipline. Michael owns
intent/vision (theme, content, balance); the agent owns the code within it.

## The shape of the thing

- **`shared/`** — the deterministic simulation, the crown jewel. Pure TypeScript,
  fixed-timestep tick. Kernels follow polyline paths (position = f(distance), NO
  pathfinding). Seeded RNG injected (`shared/rng.ts`). The whole game is a pure
  function of `(seed + command log)` — which is what makes MP cheap and high scores
  replay-verifiable. `shared/sim/smoke.ts` is the oracle.
- **`server/`** — Phase 0: static server (dist + /healthz + /version). Phase 2: adds
  the authoritative ws game server (command relay), reusing the chips/deadweight table
  harness.
- **`src/`** — Svelte 5 + canvas client. Renders the board, dispatches commands into
  the sim; in SP the sim runs client-side, in MP the server runs it.
- **`content/`** (under shared) — towers, kernels, cobs, maps, rounds as DATA. Adding
  content is authoring, not programming.

Full design: `docs/design.md`. Roadmap/record: `ROADMAP.md`.

## The discipline (inherited from first-orbit/deadweight/chips)

1. **Build the oracle first.** New sim capability gets an assertion in `smoke.ts`
   before/with the feature.
2. **Inverse hypothesis.** Include tests that MUST fail: a no-tower defense MUST leak
   all lives and lose. A test that can't fail proves nothing.
3. **Replay identity IS the oracle AND the leaderboard verifier.** Recorded command
   log replayed against the sim reproduces the final state exactly.
4. **The published artifact is the test.** `npm run build` then verify in a real
   browser (playwright-cli); dev hides prod-only crashes.
5. **`/version` is the deploy oracle.** `curl kernelpanic.cpuchip.net/version` must
   equal the pushed short-sha before a deploy is "done."

## Gates (before every commit)

```bash
npm run smoke && npm run compile && npm run build
```

For co-op (server) changes also run `npm run wstest` (start `npm run serve` first —
it spins two clients through a real WebSocket). Green, then commit. Pushing `main`
auto-deploys via Dokploy.

## Conventions

- `.npmrc` sets `legacy-peer-deps=true`; `npm ci --legacy-peer-deps`.
- **Determinism matters.** `shared/` stays pure and side-effect-free — no `Date.now()`,
  no bare `Math.random()` (RNG is injected and seeded). The only `Math.random` allowed
  is cosmetic client-only render jitter, never in the sim path.
- Currency is **butter** (earned per pop, generated by the Butter Churn). Enemies are
  **kernels**; bosses are **cobs**. Parody only — our own names/art/code, no Bloons
  assets or trademarked terms.
- Keep `ROADMAP.md` current. Workspace journal + memory live under the parent's
  `.spec/journal/` and memory (`project_kernel_panic`).
